Gaps & trade-offs pins ①–④ above · ● high ● medium ● low

● Open — worth a decision

① Harness can't see the chat pin

The Active Task is kept out of the global snapshot the background orchestrator reads (so it can't leak between contexts). A chat→background hand-off doesn't inherit it yet. Chat itself is fully covered.

② Long-chat summarizer skips your messages

On very long chats the summary condenses Clem's side + tool results but not your original asks, so why you asked can blur. The pinned list survives regardless (it lives outside the message thread).

③ A critical low-overlap fact can fall out

Recall scoping is a boost, not a filter. “Never email the test list” shares few words with “draft the Q2 campaign,” so it can rank below the cut unless pinned.

④ New facts aren't visible the same turn

Auto-captured facts commit just after the turn (available next turn). The Active Task pin is the sync exception — which is why it fixed the list bug.

● By design — intentional

Active Task expires after 6h

Stops a stale task poisoning the next one. A multi-day pause re-discovers; standing rules live in facts and don't expire.

Idle, low-value facts decay (~60d)

Keeps long-term lean. Off by default; pinned facts exempt.

Pinning is all-or-nothing

A pinned fact always injects until unpinned — no “pin for 48h.”

Lexical-only when embeddings off

Graceful degradation; semantic matches return once embeddings are enabled.

● Addressed this session

Durable Active Task pin

A stated list is held across the whole conversation & used at action time.

“ok” no longer drops memory

A casual approval keeps working memory loaded when a task is pinned.

Pointer-first big lists

Large lists become a compact reference + count — no bloat, no truncation.

Recency + on-topic recall

A fresh named list outranks a stale one (floored, no regression).

Standing rules persist

“From now on always…” saved as a durable fact across sessions.

Workers can't swap the list

Fan-out packets carry “don't substitute the parent-pinned target.”

Biggest remaining levers (your call): ① bridge the pin to background hand-offs · ② feed your messages into the long-chat summarizer · ③ auto-pin truly-critical “never” rules so recall can't bury them.

What reaches Clem each turn — chat vs harness

Chat (your surface, instructions.ts) loads the per-session file → sees the Active Task the same turn it's pinned (proven by an end-to-end test). Always-on: identity · personality · profile · pinned standing rules · MEMORY.md. Working turns add: top-12 scoped facts · data landscape · tool choices · session continuity · Working Memory incl. Active Task · goals · last-12 transcript · vault hits. A casual “ok” still gets standing facts + the Active Task.

Harness (background/fan-out, harness-context.ts) reads the global snapshot → does not see a chat session's Active Task by design (Gap ①). Workers get a tight one-item packet that carries the parent's target and may not swap it.
